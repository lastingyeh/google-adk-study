# 工作流代理 (Workflow Agents)

>
> 🔗 `資料來源`：https://google.github.io/adk-docs/agents/workflow-agents/

[`ADK 支援`: `Python` | `TypeScript` | `Go` | `Java`]

本節介紹「**工作流代理 (workflow agents)**」——**專門控制其子代理執行流程的代理**。

工作流代理是 ADK 中的專門組件，純粹設計用於**編排子代理的執行流程**。它們的主要角色是管理其他代理運行的「方式」和「時間」，定義程序的控制流。

與使用大型語言模型進行動態推理和決策的 [LLM 代理](../llm-agents.md) 不同，工作流代理基於**預定義邏輯**運作。它們根據其類型（例如：順序、平行、迴圈）決定執行順序，而無需諮詢 LLM 來進行編排本身。這產生了**確定性且可預測的執行模式**。

ADK 提供三種核心工作流代理類型，每種都實現了不同的執行模式：

<div class="grid cards" markdown>

- **順序代理 (Sequential Agents)**

    ---

    按**順序**一個接一個地執行子代理。

    → [了解更多](sequential-agents.md)

- **迴圈代理 (Loop Agents)**

    ---

    **重複**執行其子代理，直到滿足特定的終止條件。

    → [了解更多](loop-agents.md)

- **平行代理 (Parallel Agents)**

    ---

    **平行**執行多個子代理。

    → [了解更多](parallel-agents.md)

## 為什麼使用工作流代理？

當您需要明確控制一系列任務或代理的執行方式時，工作流代理至關重要。它們提供：

* **可預測性：** 執行流程根據代理類型和配置得到保證。
* **可靠性：** 確保任務始終按所需的順序或模式運行。
* **結構化：** 允許您透過在明確的控制結構中組合代理來構建複雜的程序。

雖然工作流代理以確定性的方式管理控制流，但它所編排的子代理本身可以是任何類型的代理，包括具備智慧的 LLM 代理實例。這使您能夠將結構化的流程控制與靈活、由 LLM 驅動的任務執行相結合。
