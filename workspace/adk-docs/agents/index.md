# 代理 (Agents)
🔔 `更新日期：2026-01-14`

[`ADK 支援`: `Python` | `TypeScript` | `Go` | `Java`]

在代理開發套件 (ADK) 中，**代理 (Agent)** 是一個獨立的執行單元，旨在自主行動以實現特定目標。代理可以執行任務、與使用者互動、利用外部工具，並與其他代理進行協調。

ADK 中所有代理的基礎是 `BaseAgent` 類別。它作為基本的藍圖。為了建立功能性的代理，您通常會以三種主要方式之一擴充 `BaseAgent`，以滿足不同的需求 —— 從智慧推理到結構化流程控制。

<img src="https://google.github.io/adk-docs/assets/agent-types.png" alt="ADK 中的代理類型">

## 核心代理類別

ADK 提供不同的代理類別來建構複雜的應用程式：

1. [**LLM 代理 (`LlmAgent`, `Agent`)**](llm-agents.md)：這些代理利用大型語言模型 (LLM) 作為其核心引擎，以理解自然語言、推理、規劃、生成回應，並動態決定如何進行或使用哪些工具，使其成為靈活、以語言為中心任務的理想選擇。[進一步瞭解 LLM 代理...](llm-agents.md)

2. [**工作流代理 (`SequentialAgent`, `ParallelAgent`, `LoopAgent`)**](workflow-agents/index.md)：這些專業代理以預定義、確定性的模式（順序、並行或迴圈）控制其他代理的執行流程，而不使用 LLM 進行流程控制本身，非常適合需要可預測執行的結構化流程。[探索工作流代理...](workflow-agents/index.md)

3. [**自訂代理**](custom-agents.md)：透過直接擴充 `BaseAgent` 建立，這些代理允許您實現獨特的運算邏輯、特定的控制流或標準類型未涵蓋的專業整合，滿足高度量身定制的應用需求。[發現如何建構自訂代理...](custom-agents.md)

## 選擇正確的代理類型

下表提供了高層次的比較，以幫助區分代理類型。隨著您在後續章節中更詳細地探索每種類型，這些區別將變得更加清晰。

| 功能 | LLM 代理 (`LlmAgent`) | 工作流代理 (`Workflow Agent`) | 自訂代理 (`BaseAgent` 子類別) |
| :------------------- | :---------------------------------- | :------------------------------------------ |:-----------------------------------------|
| **主要功能** | 推理、生成、工具使用 | 控制代理執行流程 | 實現獨特邏輯/整合 |
| **核心引擎** | 大型語言模型 (LLM) | 預定義邏輯（順序、並行、迴圈） | 自訂程式碼 |
| **確定性** | 非確定性（靈活） | 確定性（可預測） | 取決於實現方式，兩者皆可 |
| **主要用途** | 語言任務、動態決策 | 結構化流程、編排 | 客製化需求、特定工作流 |

## 代理協同工作：多代理系統

雖然每種代理類型都有其獨特目的，但真正的強大之處往往來自於它們的結合。複雜的應用程式經常採用[多代理架構](multi-agents.md)，其中：

* **LLM 代理**處理智慧、基於語言的任務執行。
* **工作流代理**使用標準模式管理整體程序流程。
* **自訂代理**提供獨特整合所需的專業能力或規則。

瞭解這些核心類型是使用 ADK 建構複雜、強大 AI 應用程式的第一步。
