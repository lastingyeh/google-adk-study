# 短片製作代理提示詞說明

本目錄包含一系列為「短片製作」應用設計的代理 (Agent) 提示詞。這些代理協同工作，引導使用者從一個簡單的構思開始，逐步完成一個動畫短片。

## 核心流程

整個創作流程由一個核心的**導演代理 (Director Agent)** 來編排。它會依序呼叫其他專業代理，並在每個關鍵步驟後尋求使用者的批准，確保最終成品符合使用者的期望。

流程如下：

1.  **故事生成 (Story Generation)**：導演代理首先會請使用者提供故事靈感，然後呼叫**故事代理 (Story Agent)** 來創作一個完整的營火故事。
2.  **劇本創作 (Screenplay Creation)**：故事獲得批准後，**劇本代理 (Screenplay Agent)** 會將故事改編成標準的劇本格式。
3.  **分鏡視覺化 (Storyboard Visualization)**：劇本完成後，**分鏡代理 (Storyboard Agent)** 會為劇本中的每個場景生成黑白漫畫風格的分鏡圖。
4.  **影片生成 (Video Generation)**：最後，**影片代理 (Video Agent)** 會根據劇本和分鏡圖，生成最終的影片片段。

---

## 各代理詳細說明

### 1. Director Agent (`director_agent.txt`)

-   **角色**：專案總指揮、流程協調者。
-   **核心任務**：
    -   歡迎使用者並收集初步的故事構思（如場景、角色、語調）。
    -   按照「故事 -> 劇本 -> 分鏡 -> 影片」的順序，依序呼叫各個子代理。
    -   在每個步驟完成後，向使用者展示成果（故事、劇本等）並請求批准。
    -   如果使用者不滿意，它會根據回饋要求相應的代理重新生成，直到使用者滿意為止。
    -   管理並傳遞各階段的產出（例如，將批准後的故事傳遞給劇本代理）。

### 2. Story Agent (`story_agent.txt` & `story_agent_desc.txt`)

-   **角色**：一位親切、善於講故事的「童軍團長」。
-   **核心任務**：
    -   以「童軍團長」的口吻，將使用者的構思發展成一個具有完整結構（開端、發展、高潮、結局）的營火故事。
    -   在故事結尾處，提供所有角色的外貌描述，以確保後續視覺生成的一致性。
    -   `story_agent_desc.txt` 檔案則定義了何時該將使用者的請求路由給這位「童軍團長」代理，它適合需要創意和引人入勝答案的場景。

### 3. Screenplay Agent (`screenplay_agent.txt`)

-   **角色**：專業編劇。
-   **核心任務**：
    -   接收「故事代理」產出的故事和角色描述。
    -   將故事轉換為標準劇本格式，包含場景標題 (Sluglines)、動作描述 (Action Lines)、角色名稱和對話。
    -   它被特別指示要**排除**故事的框架（例如，童軍在營火旁講故事的場景），專注於核心敘事本身。

### 4. Storyboard Agent (`storyboard_agent.txt`)

-   **角色**：分鏡師。
-   **核心任務**：
    -   接收劇本和角色描述。
    -   為劇本中的**每一個場景**生成一張分鏡圖。
    -   它會呼叫 `storyboard_generate` 工具，並傳入詳細的提示詞，要求生成「**黑白漫畫書繪圖、分鏡腳本、草圖或線條畫**」風格的影像，以確保風格統一。
    -   最終輸出一系列按場景順序排列的分鏡圖連結。

### 5. Video Agent (`video_agent.txt`)

-   **角色**：影片製作人。
-   **核心任務**：
    -   接收劇本、角色描述以及對應每個場景的分鏡圖連結。
    -   為**每一個場景**生成一段影片片段。
    -   它會呼叫 `video_generate` 工具，結合劇本的動作描述、對話以及分鏡圖的視覺構圖來生成影片。
    -   最終輸出一系列按場景順序排列的影片片段連結，完成整個短片的製作。
